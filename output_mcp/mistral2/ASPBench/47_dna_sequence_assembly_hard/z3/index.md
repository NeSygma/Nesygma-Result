#### Entities and Variables
- **Fragments**: 9 fragments (F0 to F8) with given sequences.
- **Contigs**: A list of contigs, each with:
  - A unique `contig_id`.
  - A list of `fragments` (subset of F0-F8).
  - A list of `orientations` for each fragment ("forward" or "reverse").
  - A reconstructed `sequence` for the contig.
- **Excluded Fragments**: A list of `chimeric` fragments (subset of F0-F8).

#### Domains and Sorts
- **Fragment IDs**: Finite domain (0 to 8).
- **Orientations**: Boolean or Enum (`True` for "forward", `False` for "reverse").
- **Contig IDs**: Finite domain (0 to N-1, where N is the number of contigs).
- **Sequences**: Strings (will be constructed from fragments and orientations).
- **GC-content**: Calculated for each fragment to determine overlap requirements.

#### Constraints
1. **Fragment Orientation**:
   - Each fragment can be used in either "forward" or "reverse-complement" orientation.
   - Reverse-complement: Reverse the sequence and replace A↔T, C↔G.

2. **Overlap Requirements**:
   - Adjacent fragments in a contig must overlap by at least:
     - 4 bases if both fragments have GC-content > 50%.
     - 3 bases otherwise.
   - Overlap is defined as the suffix of the first fragment matching the prefix of the second fragment.

3. **Start Codon**:
   - The first fragment in each contig must start with "ATG" in its chosen orientation.

4. **Stop Codon**:
   - The last fragment in each contig must end with "TAA", "TAG", or "TGA" in its chosen orientation.

5. **Fragment Usage**:
   - Each fragment must be either:
     - Included in exactly one contig, or
     - Marked as chimeric (excluded).

6. **Chimeric Detection**:
   - Fragments that cannot fit into valid contigs must be marked as chimeric.

#### Objective
- Minimize the number of contigs.

### Phase 2: Design and Write

#### Approach
1. **Precompute GC-content** for each fragment to determine overlap requirements.
2. **Define reverse-complement** function for fragments.
3. **Model contigs** as sequences of fragments with orientations.
4. **Enforce constraints**:
   - Start codon for the first fragment in each contig.
   - Stop codon for the last fragment in each contig.
   - Overlap between adjacent fragments.
5. **Ensure fragment usage**:
   - Each fragment is used in at most one contig or marked as chimeric.
6. **Minimize the number of contigs**.

#### Z3 Implementation
- Use `Int` for fragment IDs, contig IDs, and orientations.
- Use `Bool` for chimeric flags.
- Use `String` for sequences (though Z3 strings are limited; we will construct sequences as strings in Python and assert constraints on them).
- Use `Solver` for constraints and `Optimize` for minimizing the number of contigs.

#### Notes
- Z3 does not natively support string manipulation, so we will construct sequences in Python and assert constraints on them.
- We will use `solver.add()` to enforce constraints on sequences and overlaps.
- We will use `Optimize` to minimize the number of contigs.
